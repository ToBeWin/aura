# Aura 使用指南

## 🎯 核心功能

Aura 将口语化的、带有填充词的文本转换为专业、结构化的输出。

### 支持的输出格式

1. **Email (邮件)** - 自动添加主题、称呼、正文、结尾
2. **Weekly Report (周报)** - 结构化的工作总结
3. **Social Media (社交媒体)** - 简洁有趣的文案
4. **Code Comment (代码注释)** - 专业的代码文档
5. **Auto Detect (自动检测)** - AI 自动识别最合适的格式

### 支持的语气风格

- **Professional (专业)** - 正式、严谨
- **Casual (随意)** - 轻松、自然
- **Friendly (友好)** - 温暖、亲切
- **Formal (正式)** - 庄重、规范

## 📝 使用流程

### 方式 1: 文本输入

1. 在左侧文本框输入原始文本
2. 选择输出格式和语气
3. 点击 "Refine Text"
4. 查看右侧精炼结果
5. 点击 "Copy" 复制结果

### 方式 2: 音频上传（即将推出）

1. 点击 "📁 Upload Audio" 按钮
2. 选择音频文件（支持 WAV, MP3, M4A 等）
3. 等待转录完成
4. 自动填充到输入框
5. 继续选择格式和语气进行精炼

## 💡 实用技巧

### 1. 快速去除填充词

Aura 自动识别并移除常见填充词：
- 中文: 呃、嗯、啊、那个、这个、就是、然后、嘛
- 英文: uh, um, like, you know, I mean

### 2. 自动格式推断

如果不确定用什么格式，选择 "Auto Detect"，Aura 会根据内容自动选择：
- 包含"会议"、"准备"等词 → Email
- 包含"本周"、"下周"等词 → Weekly Report
- 包含"今天"、"分享"等词 → Social Media
- 包含"函数"、"参数"等词 → Code Comment

### 3. 语气调整

同样的内容，不同语气会产生不同效果：

**原文**: "明天开会记得带文件"

- Professional: "请您务必携带相关文件参加明天的会议。"
- Casual: "明天开会别忘了带文件哦"
- Friendly: "明天开会记得带上文件呀，别忘了～"
- Formal: "敬请阁下于明日会议时携带相关文件。"

## 📊 输出质量指标

每次精炼后，Aura 会显示：
- **Confidence (置信度)**: 0-100%，表示输出质量
- **Time (处理时间)**: 秒，表示处理速度

高置信度 (>80%) 表示：
- 成功应用了个性化规则
- 格式转换准确
- 语义保留完整

## 🎨 使用场景

### 场景 1: 会议记录整理

**原始录音转文字**:
```
嗯，今天会议主要讨论了三个问题，第一个是那个，产品发布时间，大家觉得下个月可以，第二个呢是预算问题，财务说没问题，第三个是人员安排，HR 会跟进
```

**精炼后 (Weekly Report)**:
```
会议纪要

讨论议题：
1. 产品发布时间 - 确定为下个月
2. 预算审批 - 财务部门已批准
3. 人员安排 - 人力资源部跟进中
```

### 场景 2: 快速发邮件

**口述内容**:
```
跟张总说一下，那个项目进度报告我已经完成了，明天上午发给他
```

**精炼后 (Email)**:
```
主题：项目进度报告已完成

张总：

项目进度报告已完成，将于明天上午发送给您。

此致
敬礼
```

### 场景 3: 社交媒体分享

**随意想法**:
```
今天去了新开的那个咖啡店，环境不错，咖啡也好喝，推荐大家去试试
```

**精炼后 (Social Media)**:
```
今天打卡新开的咖啡店 ☕️ 环境超赞，咖啡也很好喝！强烈推荐给爱喝咖啡的朋友们～ #咖啡 #探店
```

## 🆘 故障排除

### 问题: 初始化失败

**解决方案**:
1. 确保 Ollama 正在运行: `ollama serve`
2. 确保模型已下载: `ollama pull qwen3.5:2b`
3. 检查端口 11434 是否可用

### 问题: 处理速度慢

**解决方案**:
1. 使用更小的模型: `qwen3.5:0.8b`
2. 减少输入文本长度
3. 关闭其他占用 CPU 的应用

### 问题: 输出质量不佳

**解决方案**:
1. 尝试不同的语气设置
2. 手动指定输出格式（不用 Auto）
3. 提供更清晰的输入文本
4. 使用更大的模型: `qwen3.5:9b`

### 问题: 音频上传失败

**解决方案**:
1. 确保音频文件格式支持（WAV, MP3, M4A）
2. 确保文件大小 < 100MB
3. 检查 ffmpeg 是否安装: `which ffmpeg`
