# Aura 测试指南

## 🚀 快速测试

### 启动应用

```bash
cd aura
./start.sh
```

或手动启动：
```bash
# 终端 1: 启动 Ollama
ollama serve

# 终端 2: 启动 Aura
cd aura
npm run tauri dev
```

## ✅ 功能测试清单

### 1. 基础功能测试

#### 测试 1.1: 应用启动
- [ ] 应用成功启动
- [ ] 显示 "Aura" 标题
- [ ] 显示 "Initializing..." 状态
- [ ] 初始化完成后状态消失

#### 测试 1.2: UI 元素
- [ ] 格式选择下拉框显示正常
- [ ] 语气选择下拉框显示正常
- [ ] 输入文本框可编辑
- [ ] 输出区域显示占位符
- [ ] "Refine Text" 按钮显示正常

### 2. 文本精炼测试

#### 测试 2.1: 中文口语转邮件

**输入**:
```
呃，小李啊，那个...明天上海那个会，你帮我准备个 PPT
```

**操作**:
1. 粘贴输入到左侧文本框
2. 选择格式: Email
3. 选择语气: Professional
4. 点击 "Refine Text"

**验证**:
- [ ] 处理按钮变为 "Processing..."
- [ ] 2-5 秒后显示结果
- [ ] 输出文本格式正确（有主题、称呼、正文）
- [ ] 填充词（呃、那个）被移除
- [ ] 显示置信度 (>70%)
- [ ] 显示处理时间

#### 测试 2.2: 英文口语转社交媒体

**输入**:
```
uh so like today I went to this amazing coffee shop you know and um the latte was really good
```

**操作**:
1. 输入文本
2. 选择格式: Social Media
3. 选择语气: Casual
4. 点击 "Refine Text"

**验证**:
- [ ] 填充词（uh, like, um, you know）被移除
- [ ] 输出简洁有趣
- [ ] 可能包含 emoji
- [ ] 语气轻松随意

#### 测试 2.3: 自动格式检测

**输入**:
```
这周主要做了三件事，修 bug、开会、写文档
```

**操作**:
1. 输入文本
2. 选择格式: Auto Detect Format
3. 选择语气: Professional
4. 点击 "Refine Text"

**验证**:
- [ ] 自动识别为 Weekly Report 格式
- [ ] 输出结构化（列表或段落）
- [ ] 内容完整准确
